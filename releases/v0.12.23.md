# v0.12.23

## New Features 🎉

* **Add blob store support** ([#3606](https://github.com/pluralsh/console/pull/3606)) by @michaeljguarino

  > Blob store support has been added, enhancing the Plural Console's capabilities for managing and storing large binary objects. This feature allows users to efficiently handle data such as images, videos, and other files, streamlining workflows that require extensive data storage and retrieval. With this integration, users can leverage the blob store for improved data management within their Kubernetes environments.

* **Compatibility Matrix Update 2026-05-24** ([#3607](https://github.com/pluralsh/console/pull/3607)) by @app/github-actions

  > The compatibility matrix for the EKS add-ons has been updated to include new versions for the Datadog Operator and Kubecost, specifically v0.1.26-eksbuild.1 and v3.1.73-eksbuild.1, respectively. These updates ensure compatibility with Kubernetes versions 1.28 through 1.35, enhancing user experience by providing support for the latest Kubernetes releases and improving operational reliability.

* **feat(controller): add WorkbenchPrompt CRD and controller** ([#3611](https://github.com/pluralsh/console/pull/3611)) by @floreks

  > The new WorkbenchPrompt Custom Resource Definition (CRD) and its associated controller have been added, enabling users to create and manage saved prompts for workbenches within the Plural Console. This feature enhances the user experience by allowing for better organization and retrieval of prompts, facilitating more efficient workflows in Kubernetes management.

* **feat: Update saving prompts and dropdown** ([#3612](https://github.com/pluralsh/console/pull/3612)) by @maciaszczykm

  > The update enhances the user experience for managing saved prompts within the workbench by introducing a new dropdown interface that allows for easier selection and synchronization of prompts. Users can now quickly select saved prompts, which will automatically populate the input field, streamlining the workflow for creating jobs. Additionally, a new BookmarkAddIcon has been added to improve the visual design of the saved prompts feature, making it more intuitive and user-friendly.

* **feat: Implement chatbot features** ([#3613](https://github.com/pluralsh/console/pull/3613)) by @maciaszczykm

  > New chatbot features have been implemented, allowing users to trigger workbench runs through chat platforms. This enhancement streamlines workflows and provides an experimental interface for integrating chatbots into the Plural ecosystem, improving automation and user interaction within the workbench environment. Additionally, several UI elements have been updated to reflect successful actions, enhancing user feedback during operations.

* **Compatibility Matrix Update 2026-05-26** ([#3614](https://github.com/pluralsh/console/pull/3614)) by @app/github-actions

* **Compatibility Matrix Update 2026-05-26** ([#3615](https://github.com/pluralsh/console/pull/3615)) by @app/github-actions

Users will benefit from improved performance and reduced complexity when deploying and managing containers within their Kubernetes clusters.

Additionally, a new setup guide for integrating Slack chatbots has been added, providing users with detailed instructions on configuring app-level and bot user OAuth tokens for optimal functionality. These changes aim to streamline user experience and improve the effectiveness of AI-driven features.

## Bug Fixes 🐛

* **fix: prevent stale ready status after stack update** ([#3610](https://github.com/pluralsh/console/pull/3610)) by @zreigz

  > The update addresses an issue where the status of infrastructure stacks could remain stale after an update, potentially leading to confusion about the actual state of the stack. By refining the logic for setting the "ready" condition, the system now accurately reflects changes in stack status, ensuring users receive timely and correct information about their infrastructure. This improvement enhances the reliability of stack management and reduces the risk of operational errors.

* **fix: Add missing assignment** ([#3623](https://github.com/pluralsh/console/pull/3623)) by @maciaszczykm

  > The configuration manager now includes assignments for `maxStackRunJobs` and `maxAgentRunPods`, ensuring that these parameters are properly set when configuring the deployment operator. This fix enhances the operator's ability to manage resource allocation effectively, improving overall deployment performance and stability.

* **fix: update downloadServiceManifests tool definition** ([#3628](https://github.com/pluralsh/console/pull/3628)) by @zreigz

  > The `downloadServiceManifests` tool has been updated to provide clearer instructions on how to inspect the Kubernetes manifests applied for a `ServiceDeployment` or `GlobalService`. Users can now easily identify the required parameters, including the cluster handle and service name, ensuring a more intuitive experience when accessing and analyzing their service configurations. This enhancement improves the usability of the tool, making it easier to troubleshoot and manage Kubernetes deployments effectively.
