# v0.12.17

## New Features 🎉

* **add wiz-agent scraper** ([#3436](https://github.com/pluralsh/console/pull/3436)) by @kinjalh

  > The new "wiz-agent" scraper has been added, enhancing compatibility information for the Wiz ecosystem within the Plural Console. This update includes support for the Wiz Admission Controller, Wiz Network Analyzer, and Wiz Sensor, allowing users to easily access compatibility details and Helm chart versions for these applications. This integration streamlines the deployment process and ensures users can effectively manage their Kubernetes environments with the latest tools from Wiz.

* **chore(cloud-query): making cloud-query prod ready** ([#3458](https://github.com/pluralsh/console/pull/3458)) by @floreks

  > The cloud-query feature has been enhanced for production readiness, with significant updates to its deployment configuration and AWS integration. New environment variables for AWS configuration files have been added, allowing for better management of AWS credentials, while adjustments to the readiness probe ensure improved reliability during deployment. These changes are crucial for users looking to leverage cloud-query in a robust, production environment, enhancing both security and operational efficiency.

* **infracost integration (backend)** ([#3460](https://github.com/pluralsh/console/pull/3460)) by @kinjalh

  > The integration of Infracost into the backend now allows users to view detailed cost information for resources associated with their Kubernetes stacks. This enhancement provides insights into hourly and monthly costs, enabling better budget management and resource allocation. By incorporating Infracost resources into the stack and stack run data models, users can make more informed decisions regarding their infrastructure expenses directly within the Plural Console.
