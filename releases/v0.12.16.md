# v0.12.16

## New Features 🎉

* **feat: Add `WorkbenchCron` and `WorkbenchWebhook` CRDs, update `Workbench` and `WorkbenchTool` CRDs** ([#3453](https://github.com/pluralsh/console/pull/3453)) by @maciaszczykm

  > The new `WorkbenchCron` and `WorkbenchWebhook` Custom Resource Definitions (CRDs) have been introduced, enhancing the Plural platform's capabilities for managing scheduled tasks and webhook integrations within workbenches. This addition allows users to automate prompt executions on a defined schedule and respond to events via webhooks, streamlining workflows and improving operational efficiency. Additionally, existing `Workbench` and `WorkbenchTool` CRDs have been updated to support these new features, ensuring a cohesive experience in managing Kubernetes resources.

* **whitelist workspace field for mm** ([#3464](https://github.com/pluralsh/console/pull/3464)) by @JohnBlackwell

  > The workspace field has been whitelisted for the monitoring and logging features in both the Elastic and OpenSearch providers. This enhancement allows users to include the custom field in their log queries, improving the flexibility and specificity of log data retrieval. As a result, users can better manage and analyze their Kubernetes environments by leveraging this additional context in their logs.

* **deprecate agent sessions in UI in favor of workbenches** ([#3466](https://github.com/pluralsh/console/pull/3466)) by @jsladerman

  > The user interface has transitioned from using agent sessions to workbenches, streamlining the experience for managing AI interactions. This change enhances usability by consolidating functionalities into workbenches, allowing users to more effectively manage their AI tools and workflows in a cohesive environment. Users will now benefit from a more organized and intuitive interface, improving their overall efficiency when working with AI features.

* **Compatibility Matrix Update 2026-04-22** ([#3467](https://github.com/pluralsh/console/pull/3467)) by @app/github-actions

  > The compatibility matrix has been updated to include new versions for several key components, including Argo CD (version 3.3.8), AWS EBS CSI Driver (version 1.59.0), and Kube Prometheus Stack (version 83.7.0). These updates enhance integration with Kubernetes versions 1.20 to 1.35, improve operational workflows, and introduce critical features such as Server-Side Apply capabilities in Argo CD, ensuring users benefit from the latest enhancements and fixes in their deployments.

* **Add Canvas generation agent** ([#3468](https://github.com/pluralsh/console/pull/3468)) by @michaeljguarino

  > The addition of the Canvas generation agent enhances the Plural Console by enabling users to create visual representations of workflows and processes directly within the platform. This feature streamlines the integration of various tools and services, allowing for improved clarity and efficiency in managing Kubernetes deployments. Users can now leverage this capability to better visualize and manage their infrastructure, ultimately leading to more effective decision-making and operational oversight.

* **feat: Workbench improvements** ([#3469](https://github.com/pluralsh/console/pull/3469)) by @maciaszczykm

  > The recent workbench improvements enhance the user experience by introducing a more intuitive search and filter functionality for workbench connectors. Users can now easily filter tools by categories and types, as well as perform searches with a responsive input field, making it simpler to find and manage integrations within their development stack. These enhancements streamline workflows and improve accessibility to the tools needed for effective Kubernetes management.

* **feat(client): Enhance Workbench and WorkbenchTool fragments with new fields and observability configuration** ([#3470](https://github.com/pluralsh/console/pull/3470)) by @maciaszczykm

  > The Workbench and WorkbenchTool fragments have been enhanced to include new fields for read and write bindings, as well as an observability configuration. This update allows users to better manage permissions and integrate observability features, improving the overall functionality and monitoring capabilities of their Kubernetes workflows.

* **feat: extend AgentRunFragment** ([#3471](https://github.com/pluralsh/console/pull/3471)) by @zreigz

  > The AgentRunFragment has been extended to include two new fields: `babysit` and `babysitInterval`. This enhancement allows users to specify whether to enable babysitting for agent runs and to set an interval for this feature, providing greater control and flexibility in managing agent executions.

* **extend AgentRuntFragment** ([#3472](https://github.com/pluralsh/console/pull/3472)) by @zreigz

  > The AgentRunFragment has been enhanced to include a new optional field for "branch," allowing users to specify the branch associated with an agent run. This addition improves the granularity of agent run configurations, enabling better tracking and management of workflows across different branches in version control systems.

* **Compatibility Matrix Update 2026-04-23** ([#3473](https://github.com/pluralsh/console/pull/3473)) by @app/github-actions

  > The compatibility matrix for EKS has been updated to include new versions for several drivers and components, such as the AWS EBS CSI Driver and CoreDNS, now supporting compatibility with Kubernetes versions 1.30 through 1.35. This enhancement ensures users can leverage the latest features and improvements in these components while maintaining compatibility with their existing Kubernetes clusters, thereby improving operational efficiency and reliability.

* **Compatibility Matrix Update 2026-04-23** ([#3474](https://github.com/pluralsh/console/pull/3474)) by @app/github-actions

  > The compatibility matrix has been updated to reflect the latest chart versions and enhancements for several components, including Argo CD, Argo Workflows, and the Kubescape Operator. Notably, the Vector integration now supports a new `windows_event_log` source and improved Azure Blob authentication, alongside critical updates to its observability API that require client adjustments. These changes ensure users benefit from enhanced functionality and compatibility with the latest Kubernetes versions, improving overall system performance and reliability.

* **feat(client): add workbench, webhook, and issueWebhook fields to WorkbenchCron and WorkbenchWebhook fragments** ([#3475](https://github.com/pluralsh/console/pull/3475)) by @maciaszczykm

  > New fields have been added to the `WorkbenchCron` and `WorkbenchWebhook` fragments, including `workbench`, `webhook`, and `issueWebhook`. This enhancement allows users to better manage and configure webhooks and cron jobs within the Plural platform, improving integration capabilities and providing more granular control over automation workflows.

* **Compatibility Matrix Update 2026-04-24** ([#3476](https://github.com/pluralsh/console/pull/3476)) by @app/github-actions

  > The compatibility matrix for EKS has been updated to include support for version '1.35' across multiple existing EKS builds, enhancing the flexibility and options available for users managing their Kubernetes environments. This update ensures that users can leverage the latest version while maintaining compatibility with previous versions, facilitating smoother upgrades and deployments.

* **Compatibility Matrix Update 2026-04-24** ([#3477](https://github.com/pluralsh/console/pull/3477)) by @app/github-actions

  > The compatibility matrix has been updated to include new versions for several operators, including the kube-prometheus-stack (version 84.0.0), Argo Workflows (version 4.0.5), and OpenTelemetry Operator (version 0.149.0). Notably, the kube-prometheus-stack now features a major update to the Grafana subchart, which may introduce breaking changes that require users to validate their Grafana configurations. These updates ensure users have access to the latest features and improvements, enhancing the overall stability and functionality of their Kubernetes deployments.

* **Compatibility Matrix Update 2026-04-25** ([#3478](https://github.com/pluralsh/console/pull/3478)) by @app/github-actions

  > The compatibility matrix has been updated to include new versions and dependencies for several key components, including the kube-prometheus-stack, Argo Workflows, and GPU Operator. Notably, kube-prometheus-stack now supports version 84.0.1, which features a major Grafana subchart upgrade that may introduce new behaviors. These updates ensure users have access to the latest enhancements and fixes, improving overall system stability and performance.

* **Compatibility Matrix Update 2026-04-26** ([#3479](https://github.com/pluralsh/console/pull/3479)) by @app/github-actions

Notably, External Secrets v2.4.0 introduces features such as leader election support for high availability and enhanced secret fetching capabilities, while Kube Prometheus Stack v84.1.0 adds an optional setting to override the kube-apiserver scrape job name, improving integration with existing monitoring setups. These updates enhance functionality and provide users with greater control over their Kubernetes deployments.
