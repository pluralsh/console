# v0.12.21

## New Features 🎉

* **migrated from bamboo to swoosh** ([#3527](https://github.com/pluralsh/console/pull/3527)) by @kinjalh

  > The migration from Bamboo to Swoosh for email handling enhances the email delivery system within the Plural Console by leveraging Swoosh's capabilities. This change not only streamlines the configuration and delivery processes but also improves test assertions for email events, ensuring more reliable email functionality. Users can expect better performance and maintainability in email-related features as a result of this update.

* **feat(cloud-query): integrate InvokeLambda API for serverless function invocation** ([#3528](https://github.com/pluralsh/console/pull/3528)) by @floreks

  > The integration of the `InvokeLambda` API allows users to invoke serverless functions across AWS Lambda, GCP Cloud Run, and Azure Functions directly from the Plural Console. This enhancement streamlines the process of executing cloud functions by utilizing canonical identifiers and cloud connection credentials, thereby simplifying serverless function management and improving operational efficiency for developers.

* **Add NetworkPolicy for cloud-query component ingress restriction** ([#3543](https://github.com/pluralsh/console/pull/3543)) by @app/plural-copilot

  > A new NetworkPolicy has been introduced for the cloud-query component, restricting ingress traffic to only allow connections from console pods within the same namespace. This enhancement improves security by ensuring that only authorized components can communicate with cloud-query, thereby reducing the risk of unauthorized access and potential vulnerabilities in your Kubernetes environment. Users can enable this feature by adjusting the `networkPolicy.enabled` setting in their configuration.

Users can now access these manifests directly from the command line, enhancing their ability to manage and audit their Kubernetes resources effectively.

## Bug Fixes 🐛

* **fix(cloud-query): use forked steampipe plugin versions to fix critical vulnerabilities** ([#3549](https://github.com/pluralsh/console/pull/3549)) by @floreks

  > The cloud-query feature has been updated to utilize forked versions of the Steampipe plugins, addressing critical vulnerabilities by upgrading to specific, more secure versions of the AWS, Azure, and GCP plugins. This change enhances the security of cloud queries, ensuring users can safely interact with cloud resources without exposing themselves to known vulnerabilities. Additionally, the download script has been modified to pull plugins from the Plural repository, further reinforcing security and reliability.

* **fix: UI improvements** ([#3550](https://github.com/pluralsh/console/pull/3550)) by @maciaszczykm

  > The recent UI improvements enhance the user experience within the Plural Console's workbench components by streamlining navigation and improving the layout. Key updates include a more intuitive sidebar for managing tools, cron schedules, and webhook triggers, as well as a refined display for workbench descriptions and actions. These changes aim to facilitate easier access to essential features, ultimately improving workflow efficiency for users managing Kubernetes environments.

* **fix(stacks): prevent rendering of VirtualList when stacks are empty** ([#3553](https://github.com/pluralsh/console/pull/3553)) by @floreks

  > The rendering of the VirtualList component in the Stacks view has been updated to prevent display when there are no stacks available. This change enhances the user experience by eliminating unnecessary empty states, ensuring that users only see relevant information, especially when their search yields no results.

* **fix(nexus,agent-harness): adjust error handling in nexus proxy to align with ai provider CLIs logic** ([#3557](https://github.com/pluralsh/console/pull/3557)) by @floreks

* **fix: Fix agent release target** ([#3562](https://github.com/pluralsh/console/pull/3562)) by @maciaszczykm
