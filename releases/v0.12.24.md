# v0.12.24

## New Features 🎉

* **alertops webhook impl** ([#3627](https://github.com/pluralsh/console/pull/3627)) by @kinjalh

  > The implementation of the AlertOps webhook has been completed, allowing users to integrate AlertOps alerts into their observability workflows. This feature enables the parsing of standard alert fields and resolves associated project, cluster, and service identifiers, enhancing incident management capabilities within the Plural Console. Users can now receive and manage alerts from AlertOps seamlessly, improving their ability to respond to critical incidents.

* **feat: Stacks Terragrunt support** ([#3631](https://github.com/pluralsh/console/pull/3631)) by @maciaszczykm

  > Support for Terragrunt has been added to the Stacks feature, enabling users to manage their infrastructure as code more effectively. This enhancement allows users to leverage Terragrunt's capabilities alongside existing stack types, improving the deployment and management of complex infrastructure configurations. The integration includes updates to workflows, UI components, and the addition of a new Terragrunt logo, ensuring a seamless experience for users adopting this tool.

* **Compatibility Matrix Update 2026-05-31** ([#3640](https://github.com/pluralsh/console/pull/3640)) by @app/github-actions

  > The compatibility matrix has been updated to include new versions for several key components, enhancing support for Kubernetes clusters. Notably, the Amazon VPC CNI has been upgraded to version 1.22.1, and the kube-prometheus-stack has been refreshed to version 86.1.0, which includes routine dependency updates without introducing new top-level features. Additionally, the GPU Operator now supports version 26.3.2, ensuring compatibility with the latest Kubernetes versions and improving overall stability and performance for users.

* **Add cluster-autoscaler compatibility scraper** ([#3642](https://github.com/pluralsh/console/pull/3642)) by @hunterx-code

This update includes detailed version mappings and chart information, ensuring users can easily identify compatible versions of the cluster-autoscaler for their Kubernetes deployments. This improvement streamlines the process of managing and scaling Kubernetes clusters, making it easier to maintain optimal performance and resource utilization.

* **Streamline chatbot management** ([#3647](https://github.com/pluralsh/console/pull/3647)) by @michaeljguarino

* **Improve robustness of compat table downloads** ([#3645](https://github.com/pluralsh/console/pull/3645)) by @michaeljguarino

  > The compatibility table downloads have been enhanced for improved robustness, ensuring that add-on compatibility schemas are validated during pull requests and pushes. This change introduces a new validation workflow that checks both cloud add-on and general compatibility tables against defined JSON schemas, reducing the likelihood of errors and ensuring consistency in compatibility data. As a result, users can expect more reliable and accurate compatibility information when managing add-ons within the Plural Console.
