# v0.12.15

## New Features 🎉

* **feat(cloud-query): add Azure integration with support for metrics and logs queries** ([#3421](https://github.com/pluralsh/console/pull/3421)) by @floreks

  > Azure integration has been added to the Plural Console, enabling users to query metrics and logs from Azure Monitor. This enhancement allows for seamless monitoring of Azure resources, providing users with a unified interface to manage observability across their cloud environments. By supporting Azure, Plural further extends its capabilities in cloud observability, making it easier for teams to gain insights and maintain operational efficiency.

* **feat: Add webhook actor configuration and few other improvements** ([#3445](https://github.com/pluralsh/console/pull/3445)) by @maciaszczykm

  > Webhook actor configuration has been added, enhancing the flexibility of webhook management within the Plural Console. This update allows users to customize webhook behaviors more effectively, streamlining integration processes and improving overall user experience. Additionally, various improvements across components have been implemented, contributing to a more polished and efficient interface.

* **Compatibility Matrix Update 2026-04-19** ([#3450](https://github.com/pluralsh/console/pull/3450)) by @app/github-actions

  > The compatibility matrix has been updated to include new versions and enhancements for several components, including External Secrets, Argo CD, and the GPU Operator. Notably, External Secrets now supports additional authentication methods and has updated container images, while Argo CD's release notes detail bug fixes and minor version upgrades. These updates ensure users have access to the latest features and improvements, enhancing the overall stability and functionality of their Kubernetes deployments.

* **CloudQuery support in workbenches** ([#3451](https://github.com/pluralsh/console/pull/3451)) by @michaeljguarino

  > CloudQuery support has been added to workbenches, enabling users to integrate cloud infrastructure tools more seamlessly into their workflows. This enhancement allows for improved management of cloud connections, including AWS and Azure credentials, which streamlines the process of deploying and configuring cloud-based resources within the Plural Console. Users can now leverage this functionality to enhance their infrastructure management capabilities and improve overall efficiency.

* **feat(cloud-query): add jeager traces integration support** ([#3454](https://github.com/pluralsh/console/pull/3454)) by @floreks

  > Jaeger traces integration has been added to the Plural Console, allowing users to query distributed traces directly from Jaeger. This enhancement enables better observability and troubleshooting capabilities within Kubernetes environments, facilitating more efficient monitoring and debugging of applications. Users can now configure Jaeger as a tool in their workbench, providing a seamless experience for managing trace data alongside other observability tools.

* **add cloud connections tools to workbenches** ([#3455](https://github.com/pluralsh/console/pull/3455)) by @jsladerman

  > Cloud connection tools have been added to the workbench feature, enhancing the ability to integrate various cloud services directly within the platform. This update allows users to create and manage cloud connections more efficiently, streamlining workflows and improving overall productivity in managing Kubernetes resources. With these tools, users can now leverage cloud services seamlessly, making it easier to deploy and configure applications across different environments.

* **Compatibility Matrix Update 2026-04-21** ([#3456](https://github.com/pluralsh/console/pull/3456)) by @app/github-actions

  > The compatibility matrix for EKS has been updated to include support for version 1.35. This enhancement ensures that users can deploy and manage their applications on the latest EKS version, improving compatibility and stability within their Kubernetes environments.

* **Compatibility Matrix Update 2026-04-21** ([#3457](https://github.com/pluralsh/console/pull/3457)) by @app/github-actions

  > The compatibility matrix has been updated to include version 3.2.2 of the AWS Load Balancer Controller, which addresses Helm RBAC permissions for the Gateway API's `ListenerSet` resources. Users are advised to perform a standard `helm upgrade` to ensure their deployments are using the latest image tag and to re-apply CRDs as necessary. This update enhances stability and reduces errors in mixed Gateway API installations by improving how the controller handles missing CRDs.

* **feat: implement sentinel trigger** ([#3459](https://github.com/pluralsh/console/pull/3459)) by @zreigz

  > The implementation of the Sentinel Trigger feature introduces a new Custom Resource Definition (CRD) for managing Sentinel triggers within the Plural Console. This enhancement allows users to define and run automated checks on their Kubernetes resources, improving observability and operational efficiency by enabling proactive monitoring and response to potential issues. With this feature, users can better manage their deployments and ensure system reliability through automated oversight.

* **Fix soft serve container** ([#3461](https://github.com/pluralsh/console/pull/3461)) by @michaeljguarino

  > The soft serve container has been updated to streamline the data copying process, consolidating the data directories into a single copy command. This change simplifies the Dockerfile, ensuring that the soft serve application has access to the necessary data without redundancy, which can enhance build efficiency and reduce potential errors during deployment.


## Other Changes

These enhancements improve the overall functionality and stability of the Plural Console, ensuring a more robust experience for users managing their Kubernetes environments.
