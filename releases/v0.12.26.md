* **Compatibility Matrix Update 2026-06-14** ([#3717](https://github.com/pluralsh/console/pull/3717)) by @app/github-actions

* **feat: Convert streaming -> nonstreaming openai requests** ([#3718](https://github.com/pluralsh/console/pull/3718)) by @michaeljguarino

  > OpenAI requests have been transitioned from a streaming to a non-streaming model, enhancing the handling of LLM interactions within the Plural Console. This change simplifies the request process by ensuring that the streaming proxy is only activated when the AI proxy is enabled, improving overall system reliability and performance. Users can now expect a more straightforward integration with OpenAI services, reducing potential errors and streamlining their workflows.

* **Compatibility Matrix Update 2026-06-15** ([#3719](https://github.com/pluralsh/console/pull/3719)) by @app/github-actions

  > The compatibility matrix has been updated to include an end-of-life (EOL) date of June 1, 2026, for the KEDA chart version 2.18.0. This change is crucial for users to plan their upgrades and ensure continued compatibility with Kubernetes versions, helping to avoid potential disruptions in their deployments.

* **refactor(metrics): centralize graph rendering logic** ([#3720](https://github.com/pluralsh/console/pull/3720)) by @floreks

  > The graph rendering logic for Kubernetes metrics has been centralized, streamlining the display of resource usage data across components. This refactor enhances maintainability and performance, allowing for more consistent and efficient rendering of CPU and memory metrics in the Plural Console. Users can expect improved visualizations and a more cohesive experience when monitoring their Kubernetes resources.

* **add gateway api support for console chart** ([#3721](https://github.com/pluralsh/console/pull/3721)) by @zreigz

  > Gateway API support has been added to the Plural Console chart, allowing for enhanced routing capabilities through the introduction of HTTPRoute resources. This update enables users to define more flexible and powerful routing rules for their applications, improving the management of traffic within Kubernetes clusters. By integrating Gateway API, users can better manage their ingress configurations, facilitating a smoother deployment process and improved application performance.

* **feat: add streaming proxy for responses** ([#3723](https://github.com/pluralsh/console/pull/3723)) by @michaeljguarino

  > A streaming proxy has been added for handling responses in the Plural Console, enhancing the interaction with OpenAI's API. This feature allows for more efficient processing of responses by enabling real-time streaming, which is particularly beneficial for applications that require immediate feedback, such as chat interfaces. By improving the way responses are managed, users can expect a more responsive and dynamic experience when integrating AI functionalities into their workflows.

## Bug Fixes 🐛

* **fix: Cache Terraform plan and state** ([#3695](https://github.com/pluralsh/console/pull/3695)) by @maciaszczykm

  > Caching the Terraform plan and state has been implemented to enhance performance during deployments. This change reduces redundant calls to Terraform by storing the parsed state and plan in memory, significantly speeding up operations that require these resources. As a result, users will experience faster execution times and improved efficiency when managing their infrastructure.

* **fix(plrl-metrics): update connection field name** ([#3730](https://github.com/pluralsh/console/pull/3730)) by @floreks

  > The connection field name for Prometheus in the Plural metrics module has been updated from `prometheus_connection.url` to `prometheus_connection.host`. This change enhances clarity and consistency in the configuration settings, ensuring users can accurately specify the host address for Prometheus connections. Additionally, the `to_proto` function now accepts both `WorkbenchTool` and `ToolConnection` types, improving its flexibility and usability in handling different connection scenarios.

* **fix: Restore workbench mode selector trigger label** ([#3733](https://github.com/pluralsh/console/pull/3733)) by @maciaszczykm

  > The workbench mode selector now includes a trigger label for the 'Coding agent' mode, enhancing user clarity by providing a more descriptive interface element. This change improves the overall user experience by making it easier to understand the current mode selection at a glance, thereby streamlining the workflow for users interacting with the workbench feature.
