# Linear Webhook Setup for Plural

References:
- [Linear Developers - Webhooks](https://linear.app/developers/webhooks)
- [Linear API settings](https://linear.app/settings/api)

Linear is the one provider here where the signing secret is generated by Linear, not manually entered at creation time.

## 1. Create webhook in Linear and copy its signing secret

In Linear (`https://linear.app/`), open **Settings** -> **API** -> **Webhooks**:

1. Create a webhook (use a temporary placeholder URL if needed)
2. Select team/resource scope (usually `Issue` events at minimum)
3. Save and open webhook details
4. Copy the webhook signing secret

## 2. Create the webhook in Plural

In Plural:

1. Set **Type** to `Ticketing`.
2. Set **Provider** to `LINEAR`.
3. Enter webhook **Name**.
4. Paste the Linear signing secret as Plural **Signing secret**.
5. Click **Create new webhook**.

Copy the Plural-generated webhook URL.

## 3. Update Linear webhook URL

Back in Linear webhook settings:

1. Edit the webhook URL
2. Replace placeholder with the Plural webhook URL
3. Save

## 4. Validate

Create/update a test issue in the subscribed team and confirm in Plural:

- request accepted
- signature verification succeeds
- expected issue events appear
